### План: Тестирование структуры базы данных messenger
**Цель**  
Проверить корректность DDL-схемы и поведенческих ограничений БД.

**Объект тестирования:**  
- Таблицы: `users`, `chats`, `user_chats`, `messages`, `attachments`, `notifications`,  
- Внешние ключи (`ON DELETE`, `ON UPDATE`),  
- Ограничения: `UNIQUE`, `ENUM`, `is_deleted`,  
- Индексы, поле `last_message_at`.

**Методы:**  
- Ручной запуск self-contained SQL-скрипта (`test_db_structure.sql`),  
- Анализ вывода (все проверки возвращают `1`),  
- Верификация через `information_schema`.

**Критерии успеха:**  
- ✅ Все 13 тестов завершены со статусом PASS (`*_ok = 1`),  
- ✅ Нет ошибок при выполнении (`ERROR ...` отсутствует),  
- ✅ FK, индексы, soft-delete, ENUM работают по спецификации.

**Выход:**  
- `test_db_structure.sql` — тестовый скрипт,  
- `db_schema_test_report.md` — полный отчёт по результатам,  
- Рекомендации по сопровождению схемы (в отчёте).
