# План: API Security Testing  
**Цель** 
Выявить уязвимости, связанные с безопасностью API.  

**Объект тестирования**:  
- Все `POST`/`GET` эндпоинты, работающие с пользовательским вводом (`content`, `username`, path-параметры),  
- Заголовки CORS, `Server`, `X-Powered-By`.  

**Методы**:  
- Ручное тестирование через `curl`,  
- Анализ ответов и состояния БД,  
- Проверка существования внешних доменов (`attacker.com`, `evil.com`).  

**Критерии успеха**:  
- ✅ Нет XSS в сохраняемых данных,  
- ✅ CORS настроен безопасно (`allow-origin` ≠ `*` при `allow-credentials: true`),  
- ✅ Заголовки не раскрывают стек,  
- ✅ Ошибки БД → `4xx`, не `500`.  

**Выход**:  
- `api_security_test_report.md` — полный отчёт,  
- Рекомендации по устранению уязвимостей.
