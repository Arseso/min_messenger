# План: UI Testing (Android)

**Цель**:  
Проверить поведение мобильного приложения на Android-устройствах, включая:
- исполняется ли XSS,
- обработка ошибок при offline,
- работа WebSocket,
- отображение граничных данных (эмодзи, кириллица, длинные сообщения),
- межпользовательский чат.

**Scope**:  
- Устройства: эмулятор (Pixel 4, API 31) и/или реальное Android-устройство,  
- Сценарии: аналогичны `ui_web/`,  
- Инструменты: `flutter run`, `adb logcat`, Android Studio.

**Критерии успеха**:  
✅ Приложение запускается на Android,  
✅ XSS не исполняется (отображается как текст),  
✅ UI не крашится при ошибках сети,  
✅ Длинные сообщения и спецсимволы отображаются корректно.

**Выход**:  
- `ui_android_test_report.md` — отчёт с результатами или причиной невозможности тестирования.
